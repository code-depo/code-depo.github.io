Merge sort

  • This is a divide and conquer, recursive algorithm.
  • First divide the list into nearly equal parts.
  • recursively apply merge sort on the divided parts.
  • Merge the sorted lists.
  • Base case is when the list has zero or one element.
  • Merge is a linear time operation in which we take two sorted lists and output a sorted a list which formed by the elements of the two lists.

Complexity

  • Division of list can be done in linear time.
  • In each recursive call we divide the list into two equal parts.
  • Therefore, depth of recursion is log2(n)
  • At each level of recursion we have 2d no. of sublists d is depth of recursion, each of size n/(2d)
  • Therefore, at each level the merge operation for all the list takes O(n) i.e., O(2d*n/2d)
  • The depth goes till log2(n)
  • Hence the total time, space complexity O(nlog2(n))
